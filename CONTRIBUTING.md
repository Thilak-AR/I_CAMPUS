# Contributing to I_Campus

## Branch Rules
- `main`: protected, stable release branch  
- `dev`: integration branch for daily work  
- `feature/*`: one feature or module per branch  

## Commit Messages
Use short clear messages, e.g.  
- feat: add student dashboard  
- fix: resolve attendance bug  

## Pull Requests
- Create PR from your feature branch into `dev`
- Ensure CI tests pass
- At least one code review before merge
